# BaguetteWorkshop App

Информационная система для автоматизации работы багетной мастерской. Клиентское приложение на базе **Qt/C++** и **QML**, работающее с удаленной базой данных PostgreSQL.

## Технологический стек
* **Frontend:** QML, Qt Quick (адаптивный интерфейс)
* **Backend:** C++ 23, Qt Framework
* **Database:** PostgreSQL 16 (драйвер QPSQL, удаленный хостинг)
* **Concurrency:** QtConcurrent (асинхронные запросы к БД)
* **Monitoring:** Prometheus, Grafana, Docker Compose (мониторинг инфраструктуры)
* **CI/CD:** GitHub Actions + Self-Hosted Runner (автоматическая сборка)

## Архитектура работы с БД

Приложение спроектировано для безопасной и стабильной работы с облачной базой данных, исключая зависания графического интерфейса.

### Ключевые особенности:
* **Singleton DatabaseManager:** Централизованное управление подключениями.
* **Thread-Local Connections:** Для каждого асинхронного потока создается уникальное именованное соединение. Это предотвращает конфликты при многопоточном доступе к БД из `QtConcurrent`.
* **Security (SSL/TLS):** Подключение к базе данных осуществляется в режиме `sslmode=require`. Весь трафик между приложением и сервером зашифрован.
* **Hidden Credentials:** Пароль от базы данных не хранится в исходном коде, а безопасно передается через переменные среды операционной системы.

## Мониторинг

Для обеспечения высокой доступности и отладки производительности в проект интегрирована система мониторинга в реальном времени.

### Ключевые возможности:
* **Full-Stack Monitoring:** Сбор и визуализация метрик серверной инфраструктуры (CPU, RAM, Disk I/O) и специфичных показателей **PostgreSQL** (активные транзакции, кэширование, нагрузка на таблицы).
* **Containerized Stack:** Весь сервис мониторинга полностью изолирован и разворачивается в один клик через **Docker Compose**.

## CI/CD Пайплайн

В проекте настроена базовая непрерывная интеграция (Continuous Integration) для автоматизации рутины.

**Как это работает:**
1. При каждом `git push` в ветку `main` срабатывает **GitHub Action**.
2. Задание передается на **Self-Hosted Runner** (локальный сервер сборки).
3. Runner автоматически подтягивает зависимости, компилирует проект и проверяет его на ошибки.
4. Статус сборки в реальном времени отображается в репозитории GitHub.

## Установка и запуск

Проект использует автоматизированную сборку. Для установки готового приложения:
1. Перейдите в раздел **Releases** данного репозитория.
2. Скачайте последнюю версию установщика (`.exe` файл).
3. Запустите скачанный файл и следуйте инструкциям мастера установки.

## Основной функционал

Система представляет собой комплексное ERP-решение для управления багетной мастерской.

### Системные возможности
* **Secure Auth:** Система ролей (Администратор, Мастер производства, Продавец) с хешированием паролей алгоритмом `SHA-256`.
* **Audit Trail:** Полное асинхронное логирование всех действий пользователей в системную таблицу `event_logs`.

### Бизнес-логика и склад
* **Inventory Management:** Учет багета, фурнитуры и наборов для вышивки в реальном времени.
* **Smart Orders:** Оформление заказов на изготовление рам с использованием строгих SQL-транзакций (защита от частичной записи).
* **Automated Stock:** Автоматическое списание материалов со склада при подтверждении заказа.
* **Retail Sales:** Модуль быстрой розничной продажи сопутствующих товаров (в один клик).

### Документация и аналитика
* **PDF Receipts:** Генерация профессиональных товарных чеков с помощью `QPdfWriter` для последующей печати.
* **Business Insights:** Визуализация статистики выручки и количества заказов в графическом интерфейсе.
* **Data Portability:** Экспорт и импорт любых таблиц в универсальный формат **CSV**.
* **Backup Engine:** Создание и восстановление полных SQL-дампов базы данных прямо из графического интерфейса администратора.
