# BaguetteWorkshop App (Client Side)

Информационная система для автоматизации работы багетной мастерской. Клиентское приложение на базе **Qt/C++** и **QML**, интегрированное с отказоустойчивым кластером баз данных.

## Технологический стек
* **Frontend:** QML, Qt Quick (адаптивный интерфейс)
* **Backend:** C++ 17, Qt Framework
* **Database:** PostgreSQL 16 (драйвер QPSQL)
* **Concurrency:** QtConcurrent (асинхронные запросы к БД)
* **CI/CD:** GitHub Actions + Self-Hosted Runner (автоматическая сборка)

---

## Архитектура интеграции с БД

Приложение спроектировано для работы в высоконагруженной и отказоустойчивой среде.

### Ключевые особенности:
*  **Singleton DatabaseManager:** Централизованное управление подключениями.
*  **Thread-Local Connections:** Для каждого асинхронного потока создается уникальное именованное соединение, что исключает конфликты при доступе к БД из `QtConcurrent`.
*  **Security (SSL/TLS):** Подключение к кластеру осуществляется в режиме `sslmode=require`. Весь трафик между приложением и облаком зашифрован.
*  **Load Balancing:** Приложение подключается к **HAProxy** (порт 5000 — запись, 5001 — чтение), что позволяет прозрачно переключаться между мастером и репликой при сбоях.

---

## CI/CD Пайплайн

В проекте настроена непрерывная интеграция (Continuous Integration).

**Как это работает:**
1.  При каждом `git push` в ветку `main` срабатывает **GitHub Action**.
2.  Задание (`job: build`) передается на **Self-Hosted Runner** (локальный сервер сборки).
3.  Runner автоматически подтягивает зависимости, компилирует проект и проверяет его на ошибки сборки.
4.  Статус сборки отображается в GitHub (зеленая галочка ✅), что гарантирует стабильность кодовой базы.

## Установка и запуск

Проект поддерживает современный цикл разработки **CI/CD**. Вам не обязательно настраивать сложную среду сборки вручную.

### Для пользователей (Ready-to-Run)
Благодаря настроенному **GitHub Actions Pipeline** и **Self-Hosted Runner**, каждое изменение кода автоматически формирует готовый дистрибутив:
1. Установите в разделе Releases последнюю версию установщика
2. Следуйте инструкциям мастера установки

---

### Для разработчиков (Сборка из исходников)
Если вы планируете модифицировать проект:
1. **Инструменты:** Qt 6.x (модули `sql`, `concurrent`), компилятор MSVC 2019/2022 или GCC.
2. **Драйвер БД:** Требуется плагин `qsqlpsql` для корректной работы с PostgreSQL.
3. **Настройка:** - Откройте проект в **Qt Creator**.
   - В `DatabaseManager.cpp` проверьте параметры подключения (IP балансировщика, порт 5000).
   - Соберите проект и запустите.

## Основной функционал

Система представляет собой комплексное ERP-решение для управления багетной мастерской.

### Системные возможности
* **Auto-Provisioning:** Автоматическое развертывание структуры БД при «чистом» старте.
* **Secure Auth:** Система ролей (Админ, Продавец, Мастер) с хешированием паролей SHA-256.
* **Audit Trail:** Полное логирование действий пользователей в таблицу `event_logs`.
* **High Availability:** Прозрачное переключение на реплику при сбоях мастера (Failover).

### Бизнес-логика и Склад
* **Inventory Management:** Учет багета, фурнитуры и наборов для вышивки в реальном времени.
* **Smart Orders:** Асинхронное оформление заказов на изготовление рам через транзакции.
* **Automated Stock:** Автоматическое списание материалов со склада при подтверждении заказа.
* **Retail Sales:** Модуль быстрой розничной продажи сопутствующих товаров.

### Документация и Аналитика
* **PDF Receipts:** Генерация профессиональных товарных чеков с помощью `QPdfWriter`.
* **Business Insights:** Визуализация статистики выручки и заказов в интерфейсе.
* **Data Portability:** Экспорт и импорт любой таблицы в формат **CSV**.
* **Backup Engine:** Создание и восстановление полных SQL-дампов базы данных через GUI.
